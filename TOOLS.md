# TOOLS — 读取规则与工具使用纪律

---

## 读取原则

**先缩小范围，再下钻，不要全库扫描。**

```
用户问题
  → 判断主题（参考 knowledge/index.md）
  → 找到对应月份或主题
  → 读取 knowledge/episodes/ 下对应片段
  → 结合 SOUL.md 风格输出
```

**每次读取不超过 3 个文件。**
读多了反而失去重点，找到最相关的就够。

---

## 知识文件结构

```
knowledge/
├── index.md              # 主题 → 月份的映射表
└── episodes/             # 按月份存放的精选片段
    ├── 2023-03/
    │   └── highlights.md
    ├── 2023-04/
    │   └── highlights.md
    └── ...
```

原始完整数据在 `data/` 目录（即 HuChenFeng 原始仓库内容）。
`knowledge/` 是从原始数据里提炼出来的精选片段，优先读这里。

---

## 什么时候读原始数据

只有以下情况才去读 `data/` 下的原始文件：
- 用户问的是某个具体日期的直播内容
- knowledge/ 里找不到相关内容
- 需要验证某句话是否真实存在

---

## 禁止行为

- 不要一次性读取整个月份的所有文件
- 不要在没有真实依据时用引号标注"他说过"
- 不要修改 `data/` 下的任何原始文件
